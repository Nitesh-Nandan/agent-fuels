02
Framework Internals
pythoncore_primitives.py
from typing import Annotated, TypedDict
from langgraph.graph import StateGraph, START, END, add_messages
from langgraph.checkpoint.memory   import MemorySaver
from langgraph.checkpoint.postgres import PostgresSaver
from langgraph.prebuilt            import ToolNode

# ── 1. STATE — everything flows through this ──────────────────
def merge_results(a, b): return a + b   # custom reducer

class AgentState(TypedDict):
    messages:    Annotated[list, add_messages]   # reducer: append
    plan:        list[str]                           # reducer: overwrite
    results:     Annotated[list, merge_results]   # reducer: custom
    step:        int
    error:       str | None

# ── 2. NODES — any Python callable ───────────────────────────
def llm_node(state: AgentState):
    response = model.invoke(state["messages"])
    return {"messages": [response]}   # partial update — reducer merges

def router(state: AgentState) -> str:
    last = state["messages"][-1]
    return "tools" if last.tool_calls else "end"

# ── 3. GRAPH ASSEMBLY ────────────────────────────────────────
builder = StateGraph(AgentState)
builder.add_node("llm",   llm_node)
builder.add_node("tools", ToolNode(tools))
builder.add_edge(START,   "llm")
builder.add_edge("tools", "llm")
builder.add_conditional_edges("llm", router,
    {"tools": "tools", "end": END})

# ── 4. CHECKPOINTERS — persistence layer ─────────────────────
memory = MemorySaver()   # dev only — lost on restart
# memory = PostgresSaver.from_conn_string(DB_URI)  # production

graph = builder.compile(checkpointer=memory)

# ── 5. INVOKE with thread_id = one conversation ───────────────
config = {"configurable": {"thread_id": "user-123-session-456"}}
result = graph.invoke({"messages": [user_msg]}, config)

# ── 6. STREAM — see every step as it runs ────────────────────
for event in graph.stream({"messages": [user_msg]}, config):
    print(event)   # {node_name: {state_update}}

# ── 7. TIME TRAVEL — replay from any past checkpoint ──────────
for state in graph.get_state_history(config):
    print(state.config, state.values, state.next)

The 7 Core Patterns
01
ReAct
DEFAULT AGENT
LLM reasons → decides to call tool → tool executes → result feeds back to LLM. Loops until done. The baseline pattern for any tool-using agent.
START → [llm] ──tool_calls?──YES→ [tools] → [llm] │ NO → END
pythonreact.py
def should_continue(state) -> str:
    last = state["messages"][-1]
    return "tools" if last.tool_calls else "end"

builder.add_node("llm",   call_llm)
builder.add_node("tools", ToolNode(tools))
builder.add_edge(START, "llm")
builder.add_edge("tools", "llm")
builder.add_conditional_edges("llm",
    should_continue, {"tools":"tools", "end":END})
02
Plan-and-Execute
MULTI-STEP
Separate planner from executor. Planner creates steps upfront. Executor runs each step. Replanner adapts based on results. Better than ReAct for long-horizon tasks.
START→[plan]→[execute]→done?→YES→END ↑ NO [replan]←──────
pythonplan_execute.py
class PlanState(TypedDict):
    input:      str
    plan:       list[str]
    past_steps: Annotated[list, operator.add]
    response:   str | None

def planner(state):
    plan = planner_llm.invoke(state["input"])
    return {"plan": plan.steps}

def should_end(state) -> str:
    return "end" if state.get("response") else "execute"
03
Reflection
SELF-CRITIQUE
Agent generates output, then critiques it, then revises. Loop until quality threshold or max iterations. Use for writing, code, structured outputs.
START→[generate]→[reflect]→PASS?→YES→END ↑ NO→ revise
pythonreflection.py
def should_continue(state) -> str:
    if state["iterations"] >= 3:
        return "end"
    last = state["messages"][-1].content
    return "end" if "PASS" in last else "generate"
04
Supervisor
MULTI-AGENT
One supervisor LLM routes between specialist agents. Each specialist is a subgraph. Supervisor decides who acts next until FINISH. Best for heterogeneous tasks.
START→[supervisor]→[researcher]→back ↑ →[coder] →back └──────────→FINISH→END
pythonsupervisor.py
for member in members:
    builder.add_node(member, make_worker(member))
    builder.add_edge(member, "supervisor")

builder.add_conditional_edges("supervisor",
    lambda s: s["next"],
    {**{m:m for m in members}, "FINISH":END})
05
Swarm (Handoff)
DECENTRALISED
Agents hand off to each other directly — no central supervisor. Each agent decides if it can handle or should transfer. More autonomous than supervisor pattern.
[triage] ──handoff──→ [billing] ↑ ──handoff──→ [tech] └─────────────────────────
pythonswarm.py
from langgraph_swarm import create_swarm, create_handoff_tool

to_billing = create_handoff_tool("billing_agent")
to_tech    = create_handoff_tool("tech_agent")

swarm = create_swarm(
    [triage, billing, tech],
    default_active_agent="triage_agent"
).compile(checkpointer=memory)
06
Human-in-the-Loop
APPROVAL GATE
Graph pauses at interrupt point. Human reviews, edits state, approves. State persisted during wait — can be hours or days. Essential for high-stakes actions.
[agent]→⏸ interrupt→HUMAN REVIEWS→resume→[action]
pythonhitl.py
graph = builder.compile(
    checkpointer=memory,
    interrupt_before=["dangerous_action"]
)
# Run until interrupt
graph.invoke({"messages": [task]}, config)
# Human reviews
graph.get_state(config)
graph.update_state(config, {"plan": safer})
# Resume
graph.invoke(None, config)
07
Parallelism (Map-Reduce)
FAN-OUT / FAN-IN
Fan out to N parallel branches using Send(). Each branch runs simultaneously. Fan-in aggregates all results. Massively reduces latency for independent subtasks — N subtasks in time of 1.
pythonparallel.py
from langgraph.types import Send

# Fan-out: spawn N parallel branches dynamically from a list
def fan_out(state):
    return [Send("process_item", {"item": item})
            for item in state["items"]]   # each item = one parallel branch

# Each branch runs independently and in parallel
def process_item(state):
    result = expensive_llm_call(state["item"])
    return {"results": [result]}   # Annotated reducer collects all

# Fan-in: all results merged by reducer, then aggregated
def aggregate(state):
    summary = summarise_llm.invoke(str(state["results"]))
    return {"final_answer": summary}

builder.add_conditional_edges("router", fan_out)   # returns list of Send()
builder.add_edge("process_item", "aggregate")        # all branches converge here
04
Memory & State — Deep Dive
TYPE 1
In-Thread
Message history within one conversation. Managed by checkpointer. Persists across runs for the same thread_id.
Use: conversation history within a session
TYPE 2
Cross-Thread
Facts that persist across all conversations for a user. Stored in external KV store. Thread A can read what Thread B learned.
Use: user prefs, profile, facts across sessions
TYPE 3
Semantic
Past experiences stored as embeddings. Retrieved by cosine similarity. "What do I know relevant to this query?" — nearest neighbour search.
Use: large memory corpus, relevance-based recall
TYPE 4
Episodic
Records of past task executions: input + actions + outcome. Retrieved as few-shot examples. Agent improves automatically over time.
Use: agent learns from past task runs
TYPE 5
Procedural
System prompt updates based on learned rules. The agent's "habits" baked into the prompt. Requires a memory extraction step.
Use: persistent behavioural rules for the agent
pythonfull_memory_system.py
from langgraph.store.memory  import InMemoryStore
from langchain_openai         import OpenAIEmbeddings

# ── Cross-thread store (Types 2–5) ────────────────────────────
store = InMemoryStore(
    index={                                           # add this for semantic search
        "embed":  OpenAIEmbeddings(model="text-embedding-3-small"),
        "dims":   1536,
        "fields": ["fact"]
    }
)

# ── Save a memory (cross-thread) ─────────────────────────────
async def save_memory(user_id, fact):
    await store.aput(
        namespace=(user_id, "facts"),
        key=str(uuid4()),
        value={"fact": fact, "ts": now().isoformat()}
    )

# ── Semantic search — relevance-based retrieval ───────────────
async def search_memories(user_id, query):
    return await store.asearch(
        namespace=(user_id, "facts"),
        query=query,     # embedded + cosine similarity
        limit=5
    )

# ── Memory load node (runs at start of every invocation) ──────
async def load_memory_node(state, *, store):
    memories = await store.asearch((state["user_id"], "facts"))
    context  = "\n".join([m.value["fact"] for m in memories])
    return {"memory_context": context}

# ── Memory save node (runs at end of every invocation) ────────
async def save_memory_node(state, *, store):
    new_facts = await extractor_llm.ainvoke(state["messages"])
    for fact in new_facts:
        await save_memory(state["user_id"], fact)

# ── Compile with BOTH checkpointer + store ────────────────────
graph = builder.compile(checkpointer=postgres_cp, store=store)
#   checkpointer = in-thread memory (Type 1)
#   store        = cross-thread memory (Types 2–5)

Production Deployment
pythonproduction_patterns.py
from langgraph.pregel import RetryPolicy
import os

# ── LangSmith tracing — 3 lines, enable from day 1 ───────────
os.environ["LANGCHAIN_TRACING_V2"] = "true"
os.environ["LANGCHAIN_API_KEY"]     = "lsv2_..."
os.environ["LANGCHAIN_PROJECT"]     = "supplier-agent-prod"

# ── Retry policy on LLM nodes ─────────────────────────────────
builder.add_node("llm", llm_node,
    retry=RetryPolicy(max_attempts=3, retry_on=RateLimitError))

# ── Resume from checkpoint — no progress lost ─────────────────
# Same thread_id + crash → resumes from last successful step
graph.invoke({"messages": [msg]}, config)  # crashes at step 3
graph.invoke({"messages": [msg]}, config)  # auto-resumes from step 3

# ── Time travel — replay from any past checkpoint ─────────────
history   = list(graph.get_state_history(config))
past_conf = history[5].config
graph.invoke(None, past_conf)   # replay from step 5

# ── Tag runs for LangSmith filtering ──────────────────────────
config = {
    "configurable": {"thread_id": thread_id},
    "tags":         ["production", "v2.1"],
    "metadata":     {"user_id": uid, "feature": "enrichment"}
}
Production Checklist
✓PostgresSaver — not MemorySaver (lost on restart)
✓thread_id: user_id + session_id + feature name
✓RetryPolicy on all LLM nodes — RateLimit + ServiceUnavailable
✓Max iteration guard in every loop — counter in state
✓LangSmith tracing + tags + metadata from day 1
✓Eval suite: 50 golden test cases, run on every deploy
✓Token cost tracking — alert on per-node spend spikes
✓Checkpoint TTL — expire old threads, don't store forever
✓Per-node timeout — don't let one slow tool block everything
✓Health check — graph.get_state() on a known test thread
