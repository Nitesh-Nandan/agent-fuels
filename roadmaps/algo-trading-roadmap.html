02
The Trading Engineer Mindset
🔬
Hypothesis → Test → Measure
Every strategy is a hypothesis. You don't "believe" in it — you test it on historical data, measure its edge, and deploy only when the statistics are in your favour. Intuition is a starting point, not a trading decision.
⚙️
Rules, Not Feelings
The enemy of retail traders is discretion during execution. Your algo does exactly what the backtest says — no "but the market feels different today". If a rule needs overriding, change the rule, retest, then deploy.
📉
Risk First, Returns Second
Every trade is sized by how much you can lose, not how much you can gain. Max 1–2% of capital at risk per trade. The math of compounding means avoiding catastrophic loss matters far more than maximising any one win.
🔁
The Iteration Flywheel
Live → observe → identify drift → update strategy → backtest → paper trade → deploy. The edge degrades over time as the market adapts. Your system is never "done" — it's always in version N+1.
📊
Expectancy is Everything
Expectancy = (Win% × Avg Win) − (Loss% × Avg Loss). A 40% win rate can be highly profitable if avg win is 3× avg loss. Focus on expectancy and sample size, not on any single trade's outcome.
🛡️
Compounding Requires Survival
A 50% drawdown requires a 100% gain to recover. You are a retail investor compounding capital — your primary job is to not blow up. Small, consistent, system-generated returns beat occasional big wins every time.

Trend
EMA (9,21,50,200), SMA, DEMA, TEMA, Supertrend, Ichimoku Cloud, Parabolic SAR. Answers: which direction is price moving?

Momentum
RSI, MACD, Stochastic, ROC, Williams %R, TSI. Answers: how strong is the current move?
Use: entries, divergence signals, overbought/oversold
FAMILY 3
Volatility
ATR, Bollinger Bands, Keltner Channels, Historical Volatility, IV (from NSE options chain). Answers: how much is price moving?
Use: stop-loss sizing, breakout confirmation, regime detection
FAMILY 4
Volume
VWAP, OBV, Volume Profile, VPOC, CMF, MFI, AD Line. Answers: is price moving on conviction?
Use: intraday anchoring, breakout quality, institutional activity
FAMILY 5
Structure
Support/Resistance levels, pivot points, supply/demand zones, swing highs/lows via argrelextrema, Order Blocks. Answers: where will price react?
Use: entries near structure, targets at next structure
FAMILY 6
Oscillators
CCI, DPO, Ultimate Oscillator, Aroon, DMI/ADX. Answers: is price in a range or trending?
Use: regime filter — only trend strategies when ADX > 25
FAMILY 7
Market Breadth
Advance/Decline ratio, NIFTY PCR (Put/Call Ratio), FII/DII data, India VIX, sector rotation. Answers: is the whole market healthy?
Use: macro filter — avoid longs when breadth deteriorates

Composite / Multi-factor
Z-score ranking, custom scoring functions, factor combinations. Answers: which stock is best right now across all signals?
Use: stock selection for swing universe, portfolio ranking

6 Strategy Archetypes
01
Trend Following
SWING
Ride the direction of the dominant trend. Buy pullbacks in uptrends, short rallies in downtrends. Works in trending markets (ADX > 25). Loses in choppy sideways markets — always use regime filter.
ENTRY: Price > EMA200 + EMA9 crosses above EMA21 FILTER: ADX > 25 (trending), RSI 40–65 (not overbought) STOP: Below last swing low − 0.5×ATR TARGET: 2–3×ATR or next resistance level SIZE: 1% risk / (entry − stop) = shares
pythontrend_strategy.py
def trend_signal(df):
    signal = (
        (df['Close'] > df['ema200']) &  # above long-term trend
        (df['ema9'] > df['ema21'])   &  # short-term momentum up
        (df['ema9'].shift(1) < df['ema21'].shift(1)) &  # crossover just happened
        (df['adx'] > 25)             &  # trend is strong
        (df['rsi'].between(40, 65))       # not overbought
    )
    stop  = df['Low'].rolling(10).min() - 0.5 * df['atr']
    return signal, stop
02
Mean Reversion
SWING + INTRADAY
Price stretched too far from its mean — expect snap-back. Works in range-bound, low-ADX markets. Dangerous in strong trends. Always confirm with volume divergence — a weak bounce on low volume is not mean reversion.
ENTRY: Close < BB lower + RSI < 30 + volume < avg FILTER: ADX < 20 (ranging market, not trending) STOP: 1.5×ATR below entry TARGET: VWAP or BB middle band AVOID: If strong catalyst exists (results, news)
pythonmean_reversion.py
def mean_rev_signal(df):
    oversold = (
        (df['Close'] < df['bb_lower']) &
        (df['rsi']   < 30)              &
        (df['adx']   < 20)              &  # ranging, not trending
        (df['Volume'] < df['Volume'].rolling(20).mean())  # quiet move
    )
    target = df['vwap']   # or bb_middle = (bb_upper+bb_lower)/2
    stop   = df['Close'] - 1.5 * df['atr']
    return oversold, target, stop
03
Breakout
SWING + INTRADAY
Price breaks through a key level with volume expansion. The structural level provides confluence — this isn't random noise. Volume confirmation is non-negotiable: a breakout on weak volume is a trap, not a signal.
ENTRY: Close > resistance (argrelextrema) + vol > 1.5× avg CONFIRM: RSI > 50 + not at major round number S/R STOP: Below the broken resistance level (now support) TARGET: Height of base added to breakout point TRAP: Volume < avg on breakout = likely fake-out
pythonbreakout.py
def breakout_signal(df, resistance_levels):
    # Find nearest resistance above current price
    near_res = [r for r in resistance_levels
                if df['Close'].iloc[-2] < r < df['Close'].iloc[-2] * 1.02]
    if not near_res:
        return False
    broke  = df['Close'].iloc[-1] > near_res[0]
    vol_ok = df['Volume'].iloc[-1] > df['Volume'].rolling(20).mean().iloc[-1] * 1.5
    return broke and vol_ok
04
VWAP Intraday
INTRADAY ONLY
VWAP is the institutional benchmark for intraday price. Price above VWAP = bullish intraday bias, below = bearish. Reversion to VWAP and rejection from it are both tradeable. VWAP resets daily — intraday only.
LONG: Price > VWAP + pulls back to VWAP + bounces SHORT: Price < VWAP + rallies to VWAP + rejects STOP: 0.5×ATR beyond VWAP level TARGET: Prior intraday swing high/low TIME: 9:30–11:30 AM IST only (highest volume window)
pythonvwap_strategy.py
def vwap_bounce(df_intraday):
    # df_intraday = 5-min bars, VWAP already computed
    above_vwap = df_intraday['Close'] > df_intraday['vwap']
    pullback   = df_intraday['Low'] <= df_intraday['vwap'] * 1.001
    bounce     = df_intraday['Close'] > df_intraday['Open']  # green candle
    rsi_ok     = df_intraday['rsi'].between(45, 60)
    session    = df_intraday.index.hour.isin([9, 10, 11])    # IST 9–11 AM
    return above_vwap & pullback & bounce & rsi_ok & session
05
Opening Range Breakout (ORB)
INTRADAY ONLY
The first 15–30 min of NSE session (9:15–9:30 or 9:15–9:45 AM IST) sets the opening range. A breakout above/below with volume often sets direction for the day. Simple, robust, heavily used by retail algos.
ORB HIGH: Max of first 15-min candle(s) after 9:15 IST ORB LOW: Min of first 15-min candle(s) after 9:15 IST LONG: Close > ORB High + Volume spike STOP: ORB Low (or midpoint for tighter risk) TARGET: 1.5–2× range above ORB High
pythonorb_strategy.py
def compute_orb(df_5min):
    # df_5min index is datetime in IST
    market_open = df_5min.between_time('09:15', '09:30')
    orb_high = market_open['High'].max()
    orb_low  = market_open['Low'].min()
    orb_range = orb_high - orb_low

    after_open = df_5min.between_time('09:30', '14:00')
    long_signal  = after_open['Close'] > orb_high
    return orb_high, orb_low, orb_range, long_signal

Multi-factor Swing Rank
SWING PORTFOLIO
Score a universe of NSE stocks across multiple factors. Rank them. Trade the top N. This is how quant funds think — no single indicator decides, a composite score does. Provides diversification and reduces single-stock risk.
FACTORS: Momentum(RSI) + Trend(EMA) + Volume + Structure SCORE: Weighted sum, z-score normalised per factor UNIVERSE: NIFTY 200 filtered: ADX>20, ATR%>1.5%, Vol>5Cr SELECT: Top 5–10 stocks by composite score REBAL: Weekly (swing) or daily (aggressive swing)
pythonmulti_factor.py
from scipy import stats

def score_stock(df, w_mom=0.35, w_trend=0.35, w_vol=0.30):
    # Each factor → z-score (comparable across stocks)
    mom_z   = stats.zscore(df['rsi'])
    trend_z = stats.zscore(df['Close'] / df['ema200'])  # % above 200 EMA
    vol_z   = stats.zscore(df['Volume'] / df['Volume'].rolling(20).mean())
    return w_mom * mom_z + w_trend * trend_z + w_vol * vol_z

# Score all stocks, take top 10
scores = {sym: score_stock(dfs[sym]).iloc[-1] for sym in universe}
top10  = sorted(scores, key=scores.get, reverse=True)[:10]

Backtesting Framework
METRIC 1
Sharpe Ratio
(Mean Return − Risk-Free Rate) / Std Dev of Returns. Measures return per unit of risk. >1 acceptable, >2 good, >3 excellent. Annualise by ×√252 for daily data.
Use: primary performance metric for all strategies
METRIC 2
Max Drawdown
Largest peak-to-trough decline. The most psychologically important metric — can you live through this drawdown in real capital without abandoning the system? If not, reduce position size.
Use: sizing and capital allocation decisions
METRIC 3
Calmar Ratio
Annual Return / Max Drawdown. Better than Sharpe for retail — it shows return relative to the worst thing that can happen. >1 is good. A 20% annual return with 40% max DD is a Calmar of 0.5 — not great.
Use: compare strategies with different drawdown profiles
METHOD 1
Walk-Forward Test
Train on first 70%, optimise parameters, test on last 30%. Then slide the window. If out-of-sample performance collapses vs in-sample, you've overfit. The out-of-sample result is your true edge estimate.
Use: mandatory before any live deployment
METHOD 2
Monte Carlo
Shuffle trade returns randomly, run 1000 simulations. Shows the distribution of possible outcomes — your 5th percentile drawdown is your worst-case planning number. Real edge shows up as a clear positive expectancy distribution.
Use: sizing position size and setting kill-switch thresholds
TRAP 1
Overfitting
Adding parameters until backtest looks perfect. Red flags: too many conditions, parameters optimised to <5% tolerance, fewer than 30 trades in backtest. The curve-fit will fail on live data — guaranteed.
Simple rules with 100+ trades beat complex rules with 20
pythonbacktest_engine.py
from backtesting import Backtest, Strategy
import numpy as np

class TrendStrategy(Strategy):
    ema_fast  = 9
    ema_slow  = 21
    rsi_low   = 40
    risk_pct  = 0.01  # 1% of capital at risk per trade

    def init(self):
        self.fast = self.I(ta.ema, self.data.Close, self.ema_fast)
        self.slow = self.I(ta.ema, self.data.Close, self.ema_slow)
        self.rsi  = self.I(ta.rsi, self.data.Close, 14)
        self.atr  = self.I(ta.atr, self.data.High, self.data.Low, self.data.Close, 14)

    def next(self):
        if self.fast[-1] > self.slow[-1] and self.rsi[-1] > self.rsi_low:
            stop   = self.data.Close[-1] - 1.5 * self.atr[-1]
            risk   = self.data.Close[-1] - stop
            shares = int((self.equity * self.risk_pct) / risk)
            self.buy(size=shares, sl=stop)

bt = Backtest(df, TrendStrategy, cash=100000, commission=0.0003)
stats = bt.run()
print(stats[['Return [%]', 'Sharpe Ratio', 'Max. Drawdown [%]', '# Trades']])

# ── Walk-forward (out-of-sample) ──────────────────────────────
stats_wf = bt.optimize(
    ema_fast=range(5,15), ema_slow=range(15,30),
    maximize='Sharpe Ratio',
    constraint=lambda p: p.ema_fast < p.ema_slow
)

Live Execution Architecture
LAYER 1
Data Feed
Upstox/Kite WebSocket for real-time tick + OHLC. Convert to pandas DataFrame on each bar close. Handle reconnects automatically.
LAYER 2
Signal Engine
Compute indicators on new bar. Evaluate strategy rules. Output: BUY / SELL / HOLD + quantity + stop price.
LAYER 3
Risk Gate
Before every order: check daily loss limit, max open positions, margin available, and circuit-breaker state. Block order if any limit hit.
LAYER 4
Order Engine
Place orders via Kite/Upstox API. Handle rejections, partial fills, and GTT orders. Log every order with timestamp, price, and reason.
LAYER 5
Monitor & Alert
Live PnL dashboard. Telegram bot for fill confirmations and daily summary. Kill switch if drawdown > threshold.
pythonlive_engine.py
import asyncio
from kiteconnect import KiteConnect, KiteTicker
from upstox_client import ApiClient, Configuration

class AlgoEngine:
    def __init__(self, strategy, risk_config):
        self.strategy    = strategy
        self.risk        = risk_config
        self.kite        = KiteConnect(api_key="...")
        self.daily_loss  = 0
        self.open_trades = {}

    async def on_bar_close(self, symbol, df):
        # Layer 2: compute signal
        signal, stop = self.strategy.generate(df)
        if not signal:
            return

        # Layer 3: risk gate — block if limits hit
        if self.daily_loss < -self.risk['max_daily_loss']:
            return  # circuit breaker
        if len(self.open_trades) >= self.risk['max_positions']:
            return

        # Layer 4: place order
        price = df['Close'].iloc[-1]
        risk  = price - stop
        qty   = int((self.risk['capital'] * 0.01) / risk)

        order_id = self.kite.place_order(
            variety=KiteConnect.VARIETY_REGULAR,
            exchange=KiteConnect.EXCHANGE_NSE,
            tradingsymbol=symbol,
            transaction_type=KiteConnect.TRANSACTION_TYPE_BUY,
            quantity=qty,
            product=KiteConnect.PRODUCT_MIS,     # intraday
            order_type=KiteConnect.ORDER_TYPE_MARKET,
            stoploss=round(stop, 2)
        )
        self.open_trades[order_id] = {'symbol': symbol, 'stop': stop, 'qty': qty}
        await self.alert(f"BUY {symbol} qty={qty} stop={stop:.2f}")

    async def alert(self, msg):
        # Layer 5: Telegram notification
        import httpx
        await httpx.AsyncClient().post(
            f"https://api.telegram.org/bot{BOT_TOKEN}/sendMessage",
            json={"chat_id": CHAT_ID, "text": msg}
        )
